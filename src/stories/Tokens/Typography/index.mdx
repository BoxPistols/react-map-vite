# Typography

## About

Typographyコンポーネントは、テキストのスタイルやフォーマットを指定するために使用され、MUIではマテリアルデザインのテキストスタイルを提供しています。

## WebFont

<Typography color={theme.palette.text.secondary} variant="xl">
  {"Inter + Noto 日本語 / あ亜ABCabc1230@#$"}
</Typography>

- この環境では日本語に **Noto Sans JP** を使用しています。Googleが開発したオープンソースの日本語フォントです。[Google Fonts: Noto Sans JP](https://fonts.google.com/specimen/Noto+Sans+JP)
  - Noto Sans JPの特徴は、日本語のバランスを崩さずに、英語のフォントとの組み合わせが美しく見え、WebFontを使うことでOSに依存しない日本語の表示を実現できます。
- 英語は **Inter** を使用しています。同じくGoogleが開発したオープンソースの英語フォントです。[Google Fonts: Inter](https://fonts.google.com/specimen/Inter)
  - Interは、日本語のフォントとの組み合わせが美しく見えることも特徴です。

## Typography概要

- MUIのテキストは文字の大きさ、太さ、カラー、行間など、さまざまなスタイルをカスタマイズ可能で、汎用的に適用できます
- 特に variant オプションの命名が特徴的ですが、マテリアルデザインの Type System に基づいています[Material Dedign / The type system](https://m2.material.io/design/typography/the-type-system.html#type-scale)
- 特徴としてマークアップ構造と UI の関心事を分離しながらスタイル実装を実現することができます。
  - 基本記述方法 `<Typography variant="xxx" component="yyy">Text</Typography>`
  - variant=スタイルの適応
  - component=マークアップ構造の適応
- 実装としてマークアップ構造は意識せずとも良い設定をしており、基本的には`<Typography variant="xxx">Text</Typography>`だけで良いケースが多いです。
- カラー指定や余白の追加などもオプションで指定できるので、スタイルの変更も容易です

### 主要なAPI

Typographyコンポーネントには、頻出する以下のような主要なAPIがあります:

- <code>noWrap</code>: 改行防止
- <code>align</code>: textAlign です。"center"や"left/right"などが使えます
- <code>gutterBottom</code> Typography
  コンポーネントの直下が狭い場合、下部に一定の余白を付けます

詳細 [https://mui.com/material-ui/api/typography/](https://next.mui.com/material-ui/api/typography/)

## Variant

- Mui Typography コンポーネントは [Typography Variants](/story/tokens-typography--vatiants) にあるように、以下のような主要なバリアントを持っています：
  - **h1** - **h6** 見出し。`h1`が最も大きく、`h6`が最も小さい。
    - h1は主にサービス名やロゴのAltとしてのマークアップ用途として使う事が多く、実質h2が最も大きな実用テキストとしてとして使われる事が多いです
    - headingは全てvariantとしてはdiv扱いになり、component="h2"などを書かない限りマークアップ構造はあまり気にせず視覚に専念したデザインとして利用できます
  - **subtitle1** - サブタイトルに適したスタイル。主に h2 や h3 レベルの見出しの下に使用されます。小さな見出しには通常は使いません。
  - **subtitle2** - サブタイトルに適したもう少し小さいスタイル。主に h3-h6 レベル の下に使用されます。
  - **body1** - 本文に適したスタイル。最も一般的なスタイルで、基本的には未指定の場合のデフォルトスタイルと同じになります。
  - **body2** - 本文に適したもう少し小さいスタイル
- 以下はあまり使われることがないが、以下のようなスタイルも用意されています
  - **caption** - 小さな文字の Utilty テキストとしても使用できます
  - **overline** - 小さな文字の Utilty テキストとしても使用できます
  - **button** - ボタンのテキストスタイル

### 拡張 variant

- マテリアルデザインの Typography の命名ルールがマッチしないケースもありますので、拡張 variant を用意しています。
- 具体的な Typography variant の一覧を[Variants](/story/tokens-typography--vatiants)にまとめていまるので、そちらを参照してください。

## Tailwindとの併用

- MUIのTypographyコンポーネントは、Tailwind CSSとの併用も可能です
- Tailwind CSSのクラスを直接適用することで、MUIのTypographyコンポーネントにもスタイルを適用できます
- 以下は、MUIのTypographyコンポーネントにTailwind CSSのクラスを適用した例です

<Typography className="text-center text-2xl font-bold py-2 bg-gray-200 bg- bg-gray-200 dark:bg-gray-800 text-gray-800 dark:text-gray-200 hover:bg-gray-300 dark:hover:bg-gray-700 cursor-pointer">
  {"Example: オプション指定"}
</Typography>

```tsx
<Typography className="text-center text-2xl font-bold py-2 bg-gray-200 bg- bg-gray-200 dark:bg-gray-800 text-gray-800 dark:text-gray-200 hover:bg-gray-300 dark:hover:bg-gray-700 cursor-pointer">
  {"Example: オプション指定"}
</Typography>
```

上記のように、Tailwindのクラス設計は意味的な責務を持たない利便性の反面、クラスが増えることでスタイルの一元的な管理が難しくなることがあります。

- MUIのデフォルトのスタイルとTailwind CSSのスタイルが競合する場合があるので、適用する際は注意が必要です
- また、デザイントークンはMUIのTypographyコンポーネントのスタイルを優先するため、Tailwind CSSのスタイルを適用する際は別途カスタムスタイルを定義することをお勧めします

### Tailwind CSSの採択

- Tailwindはエンジニアにとって常に賛否両論の意見があり、プロジェクトの要件、開発者の意向やスキルによって異なりますので、適切な選択を行ってください
- 今回のプロダクトではMUIのデザイントークンを優先していますが、必要に応じてTailwindを併用することで、MUIの煩雑性の弱点を補い、より柔軟なスタイリングを実現する事を目指しています

Tailwind URL [https://tailwindcss.com/docs](https://tailwindcss.com/docs)
